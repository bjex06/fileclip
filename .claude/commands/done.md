# /done - 作業結果記録コマンド

作業結果を PROJECT.md の検証ログに記録する。

## 実行時の動作

1. 直前の作業内容を要約して表示

2. ユーザーに確認：
```
この作業の結果を記録します。

📋 作業内容: [直前の作業を要約]

結果を教えてください:
○ 一発成功
△ 動くが手動修正した
× 失敗、やり直し

問題があれば教えてください:
```

3. 回答を受けたら PROJECT.md の検証ログに追記：
```markdown
| 日付 | 指示 | 結果 | 問題 | 対処 |
|------|------|------|------|------|
| 1/4 | [作業内容] | [○/△/×] | [問題] | [対処予定] |
```

4. △ × の場合、対処方法を提案：
   - Skill の改善点
   - チェックリストへの追加
   - ルールの追加

## 使用例

```
ユーザー: /done
Claude: 
  この作業の結果を記録します。
  
  📋 作業内容: Stripeの決済フロー実装
  
  結果を教えてください:
  ○ 一発成功
  △ 動くが手動修正した
  × 失敗、やり直し

ユーザー: △ Webhookの署名検証が抜けてた

Claude:
  記録しました。
  
  | 1/4 | Stripe決済実装 | △ | Webhook署名検証漏れ | Skill改善 |
  
  💡 対処提案:
  stripe-payments/SKILL.md のチェックリストに
  「署名検証」を太字で強調することを推奨します。
  
  今すぐ Skill を修正しますか？
```
